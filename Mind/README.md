## 敬畏
```
敬畏：敬畏越多，困难越多。
引用别人的话：'
简单说吧：在现代这种多人协作开发大型软件的场景里，你不可能搞懂哪怕一个中型项目的一切细枝末节。

那好，既然没人懂，是不是我们先坐下来等着，慢慢“敬畏”着？

你只能事先尽量搞对方向，然后胆大心细、勇往直前，踏平一个个绊脚石。

要“踏平绊脚石”，就必须先行动起来，一个个坑踩过去，一堵堵墙撞过去；坑踩多了、壁碰够了，信心、水平才慢慢上去了——相反，那些裹足不前、蹲一边敬畏啊踏实啊念叨个不停的“正确的废话”，救不了任何人。

软件开发是一种工程；“工程”首先就要动起来；不动工，你就永远不知道自己会遇到什么。

工程的难，并不仅仅是在原理上，甚至并不仅仅是在技术上——你可能有几个猪队友；你的老板钱不够了；你的竞争对手抢先发布了你正开发的功能而且比你设计的好，等等等等。

“敬畏”越多，你的困难就越多。

让一个新人懂得“敬畏”的最佳途径，绝不是在他还没抬脚前一叠声的吓唬：“别动！这个很难！你不懂的！”

重复一遍：我不认为这是敬畏。这是心理阴暗，这甚至可以说是故意给人下绊子。

走路很难，会踩坑，会扭脚，会绊倒；地上还可能有铁钉有玻璃渣。

那么，你儿子一岁学走路之前，你是不是要先给他讲道理，讲到他全都懂了你才放他下地呢？

你当然不会。你会尽量把地上收拾干净，然后鼓励他多走。每摇摇晃晃多走一步，你都会由衷的欢喜。

但是换了别人学做事，怎么就那么多敬畏了呢？多说八百句“敬畏”，能让他们少踩一个坑？' 
```
